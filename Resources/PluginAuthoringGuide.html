UsageBoard 插件编写说明

插件是一个 Python 脚本。UsageBoard 会定时执行脚本，读取 stdout 中的 JSON，并把返回的用量项目渲染成进度条。

基本规则

  • 脚本建议使用 #!/usr/bin/env python3。
  • UsageBoard 不通过 shell 执行脚本，会直接调用 Python 运行 .py 文件。
  • 插件配置参数通过命令行传入，格式为 --usageboard-param KEY=value。
  • 脚本必须向 stdout 输出一个 JSON 对象。stderr 可用于调试，但不会作为用量数据解析。
  • 脚本退出码非 0、超时、stdout 不是合法 JSON，都会在对应插件卡片上显示错误。

声明配置参数

在脚本开头注释中放入固定块 UsageBoardPlugin。UsageBoard 添加插件或刷新元数据时会读取这个注释块，并在设置界面自动生成表单。

#!/usr/bin/env python3
# UsageBoardPlugin:
# {
#   "schemaVersion": 1,
#   "name": "Example",
#   "description": "查询某个服务的用量",
#   "parameters": [
#     {
#       "name": "API_KEY",
#       "label": "Api Key",
#       "type": "string",
#       "required": true,
#       "placeholder": "Service API Key"
#     },
#     {
#       "name": "PROVIDER",
#       "label": "Provider",
#       "type": "choice",
#       "required": false,
#       "defaultValue": "GLM",
#       "options": [
#         {"label": "GLM", "value": "GLM"},
#         {"label": "ZAI", "value": "ZAI"}
#       ]
#     }
#   ]
# }
# /UsageBoardPlugin

参数字段

字段 说明
name 参数键名。脚本会收到 --usageboard-param name=value。
label 设置界面显示名称；不传时使用 name。
type 支持 string、secret、integer、boolean、choice。
required 是否必填。必填项为空时插件不能启用。
placeholder 输入框占位提示。
defaultValue 添加插件时自动填入的默认值。
options choice 类型使用，每项包含 label 和 value。

读取参数

import sys

def parse_usageboard_params(argv):
    values = {}
    index = 0
    while index < len(argv):
        if argv[index] == "--usageboard-param" and index + 1 < len(argv):
            key_value = argv[index + 1]
            if "=" in key_value:
                key, value = key_value.split("=", 1)
                values[key] = value
            index += 2
        else:
            index += 1
    return values

params = parse_usageboard_params(sys.argv[1:])
api_key = params.get("API_KEY", "")

返回数据格式

插件输出的 JSON 顶层对象必须包含 schemaVersion、updatedAt 和 items。

{
  "schemaVersion": 1,
  "updatedAt": "2026-04-29T00:00:00Z",
  "items": [
    {
      "id": "requests",
      "name": "Requests",
      "used": 1200,
      "limit": 1500,
      "displayStyle": "ratio",
      "resetAt": "2026-04-29T05:00:00Z",
      "status": "normal",
      "color": "blue"
    }
  ]
}

用量项目字段

字段 说明
id 项目稳定 ID。建议只使用字母、数字、连字符或下划线。
name 界面显示名称。
used 已使用数量。
limit 总额度。进度条按 used / limit 计算，并限制在 0...1。
displayStyle percent 显示百分比，例如 80%；ratio 显示数字占比，例如 1200/1500。
resetAt ISO 8601 时间字符串，例如 2026-04-29T05:00:00Z。缺失或过期时界面显示 --。
status 支持 normal、warning、critical、unknown。
color 可选。控制进度条颜色，支持 blue、yellow、orange、red、green；不传默认蓝色。

建议插件在查询失败时仍返回一个合法 JSON，并把错误写成一个 status: "critical" 的项目；这样用户能在 UsageBoard 界面里直接看到原因。
